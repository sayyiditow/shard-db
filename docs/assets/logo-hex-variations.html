Hex shards — refinement variations

All variations on concept #1. Pick a single one or a hybrid.

A. Original

Three overlapping hex tiles, lead one bright. Where we started.

primary #58a6ffaccent #1f6feb

Honest assessment: a competent first cut. Pretty close to generic blockchain/Web3 aesthetic though — could look like any-other-distributed-tech logo.

favicon

B. With data inside

Same hex layout but the lead tile contains scattered "records" (white dots). Resolves the "could be any distributed tech" critique by literally showing data inside the shard.

Communicates: this isn't crypto/Web3 — it's a database, and each shard contains records.

favicon (dots reduced for legibility)

C. Two hexes (minimalist)

Drop one. Two interlocking hexes is a cleaner, more sophisticated mark. Subtraction makes it feel more designed and less generic.

Still reads as "more than one shard" — the differentiator is preserved.

favicon

D. Warm palette

Same shape as A but in Rust/Postgres-adjacent oranges. Warmer, less "cloud-tech blue". Stands out next to the blue CI/Scorecard badges.

primary #f78166accent #bf4220

favicon

E. Single hex with internal facets

One hex, with three lines through it splitting the interior into 6 wedges. The "shards" are inside the cell, not separate cells. More confident and less crowded.

Conceptually closer to "this thing is one database that's internally sharded" — which is exactly what shard-db is.

favicon

F. Lockup (E + wordmark)

Variation E paired with the project name in monospace. This is what would go in the docs header / social card / GitHub repo banner.

The mark + wordmark together — best of both worlds.

My take

Of these, E (single hex with internal facets) is the strongest:

  • Cleanest favicon — reads at 16px
  • The "internal sharding" reading is conceptually accurate (one DB, internally partitioned)
  • Hardest to confuse with a blockchain/distributed-thing logo because there's only one hex, not many
  • Pairs naturally with a wordmark (variation F)

If you want the "multiple shards" reading more explicit, C (two hexes) is the next strongest. B (with data dots) is the most explicit but risks looking busy.
